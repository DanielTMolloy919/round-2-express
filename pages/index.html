API Endpoints Documentation

1. Get All Items

Retrieve all items with optional filters for category and name.

Endpoint: GET /api/items

Optional Query Parameters:

  • category - Filter by category
  • name - Filter by name

Example Requests:

// Get all items curl http://localhost:3010/api/items // Filter by category curl http://localhost:3010/api/items?category=Fruit // Filter by name curl http://localhost:3010/api/items?name=Apple // Filter by both curl http://localhost:3010/api/items?category=Fruit&name=Apple

2. Create New Item

Create a new item with a name and category.

Endpoint: POST /api/items

Required Fields:

  • name - Item name
  • category - Item category

Example Request:

curl -X POST http://localhost:3010/api/items \ -H "Content-Type: application/json" \ -d '{"name": "Orange", "category": "Fruit"}'

3. Get Single Item

Retrieve a specific item by its ID.

Endpoint: GET /api/items/:id

Example Request:

curl http://localhost:3010/api/items/1

4. Update Item (Admin Only)

Update an existing item. Requires admin privileges.

Endpoint: PUT /api/items/:id

Optional Fields:

  • name - New item name
  • category - New item category

Example Request:

curl -X PUT "http://localhost:3010/api/items/1?admin=true" \ -H "Content-Type: application/json" \ -d '{"name": "Green Apple", "category": "Fruit"}'

Response Format

All successful responses will return JSON data. Here's an example item structure:

{ "id": 1, "name": "Apple", "category": "Fruit" }

Error Handling

Error responses will include a message field explaining what went wrong:

{ "message": "Error description here" }

Common HTTP status codes:

  • 200 - Success
  • 201 - Created (for POST requests)
  • 400 - Bad Request
  • 403 - Forbidden (admin access required)
  • 404 - Not Found
  • 500 - Internal Server Error
